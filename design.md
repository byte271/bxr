# BXR Design

BXR is a browser-native x86-64 machine runtime. The current source-of-truth documents are:

- [`docs/architecture.md`](docs/architecture.md): project identity, architecture boundaries, browser constraints, determinism, snapshots, CPU/device model, and debugger expectations.
- [`docs/roadmap.md`](docs/roadmap.md): public-release gates and required long-term capability tracks, including SMP, BIOS/UEFI, networking, full disk/storage, and AVX/SIMD.
- [`docs/specs/machine-profile-v1.md`](docs/specs/machine-profile-v1.md): current minimal profile draft.
- [`docs/specs/snapshot-format-v1.md`](docs/specs/snapshot-format-v1.md): current snapshot manifest/chunk draft.
- [`docs/specs/worker-protocol-v1.md`](docs/specs/worker-protocol-v1.md): current browser worker protocol draft.
- [`docs/first-20-task-status.md`](docs/first-20-task-status.md): implementation status.
